# 第二部分介绍

现在来到了应用部分—证明那些构成亚马逊企业文化的要素能够产生成果。2015年杰夫写道，"我们想成为一个规模大的公司，同时也是一台发明机器。我们想把规模带来的超凡客户服务能力与通常与创业初创企业相关联的快速行动、灵活性和风险承受心态结合起来。"[1] 正如我们将看到的那样，在第一部分枚举的实践的细致观察与亚马逊令人印象深刻的成功和突破之间存在直接联系，这些成功包括Amazon Prime和Prime Video等服务、Kindle和Alexa等硬件（Kindle也是一项服务）、Bar Raiser招聘和倒着工作等实践方法，以及Amazon Web Services和Amazon Echo和Alexa等业务。

当然，杰夫在同一封股东信中也写道，"我认为我们是世界上最好的失败之地（我们有很多练习机会！），失败和发明是不可分的孪生兄弟。要发明，你必须进行实验，如果你事先知道这将有效，那就不是实验。大多数大型组织都欢迎发明的理念，但不愿意承受必要的一系列失败实验。"因此对亚马逊而言，其成功率较低的发明，比如Fire Phone，是有价值的。同样的道理也适用于后来成功产品的未达预期的早期迭代，例如Amazon Unbox（后来演变为Prime Video），以及Amazon Auctions和zShops（后来发展为Amazon Marketplace）。这些"失败"是公司历史的重要组成部分，既是后来成功的前身，也是实验正在进行的证据。

自然而然，如果你没有预算进行发明，就不要这样做。但是，即使预算有限，如果你采取耐心和节俭的态度，你也可以随着时间的推移取得成功。体现亚马逊企业文化意味着以长期思维和客户痴迷的方式对待发明，确保领导力原则指引方向，并部署实践方法来推动执行。杰夫写道，"长期思维利用我们现有的能力，让我们做到原本无法想象的新事情。长期导向与客户痴迷相得益彰。如果我们能够识别客户需求，并能进一步形成这种需求是有意义且持久的确信，我们的方法允许我们耐心地工作多年来交付解决方案。"[2]关键词是：耐心。许多公司如果一个倡议在短短几年内没有产生他们所寻求的回报率，就会放弃。亚马逊会坚持五年、六年、七年—在此期间不断学习和改进，管理好投资—直到它获得动力和认可。

其他关键是节俭。如果你把钱花在不会改善客户体验的事情上，比如贸易展览摊位、大型团队和炫目的营销活动，你就无法长期追求发明。Amazon Music和Prime Video是我们如何多年来通过节俭来管理投资的例子：保持团队规模小、专注于改善客户体验、限制营销支出、谨慎管理利润表。一旦我们对这些产品如何能够成为数十亿美元级别的业务有了清晰的产品计划和愿景，能够让数百万乃至数亿消费者欣喜，我们就进行大额投资。多年来耐心且精心管理的投资可以产生巨大回报。

发明并不是每个问题的解决方案。例如，亚马逊成立之初，该公司并不创建自己的计算机硬件。另一方面，当我们规划电子书业务时，我们决定通过Kindle进入硬件领域。原因是：发明在差异化最重要的地方效果最好。在公司早期，为亚马逊数据中心供电的硬件并不是与客户的差异化点—创造引人入胜的在线图书购买体验才是。而对于Kindle，正如我们在第七章中所描述的，其他公司也在销售电子书，因此拥有并控制一个杰出设备的创建对我们的客户而言具有真实价值。与客户的差异化往往是发明的关键原因之一。

而昨天为真的事情今天可能不再为真。事实上，今天亚马逊确实为其数据中心生产一些计算机硬件。这是因为为AWS数据中心设计的这种专用硬件以不微不足道的方式降低了成本并提高了可靠性。这些优势可以传递给AWS客户，形式是有意义的价格下降和提供更高可靠性的服务。

当我们进行发明时，我们的长期、耐心的方法—由客户需求驱动—与更传统的"技能优先"发明方法有根本不同，在这种方法中，公司寻求与其现有技能和能力完全契合的新商业机会。虽然这种方法可能会带来回报，但它有一个根本性的问题：公司永远不会被驱动去掌握新技能和发展新能力、聘请新类型的领导者或创建不同类型的组织。亚马逊的倒着工作流程—从客户需求开始，而不是从企业需求或能力开始—通常要求在杰夫的话语中，我们"运用新的肌肉，不管那些最初的步骤感起来多么不舒服和笨拙。"

在本书的这一部分，我们查看了四个成功的亚马逊企业文化发明的关键例子—Kindle、Amazon Prime、Prime Video和Amazon Web Services (AWS)—但还有许多其他我们可以强调但未提及的，例如Fulfillment by Amazon、Amazon Echo和Alexa，以及Kindle Direct Publishing。

我们不会为其中一个失败专门写一章，但值得在这里简要提及一个。Amazon Fire Phone是一个执行良好的流程产生了失败产品的例子。Fire Phone的推出是公司曾经承担的最大新产品倡议之一。该手机的主要差异化功能是一个叫"动态透视"的功能，这是一个由手机内四个摄像头和一个定位陀螺仪产生的3D效果。超过一千人从事了创建手机3D能力所需的许多要素，以及许多较小的创新，还有会影响客户体验的更多标准功能。约30个应用程序涉及其中，用于一键客户服务、云中的免费照片存储、时钟、日历、音乐播放器、Kindle等等。

Fire Phone在2014年6月推出。在2015年8月，它被停用。发生了什么？

首先，即使使用了PR/FAQ流程，Fire Phone也没有解决一个足够重要的客户问题或创造一个显著出色的客户体验。我（比尔）还记得在2012年我第一次了解到该项目时，我想知道为什么有人会想要手机上的3D效果，尽管它很酷。这是推出当天新闻稿的一个片段：

西雅图—(美通社)—2014年6月18日—(纳斯达克：AMZN)—亚马逊今天推出了Fire，这是首款由亚马逊设计的智能手机。Fire是唯一具有动态透视和Firefly两项突破性技术的智能手机，这两项技术让你能够通过全新的视角看到世界并与之互动。动态透视使用一个新的传感器系统来响应你持有、查看和移动Fire的方式，实现其他智能手机上不可能的体验。Firefly快速识别现实世界中的事物—网络和电子邮件地址、电话号码、二维码和条形码、电影、音乐和数百万种产品，并让你在几秒内采取行动—只需按下Firefly按钮。[3]

其次，该手机以溢价销售。亚马逊的指导原则之一是节俭，我们已经向世界展示了我们是成本效益的、打破商业模式的公司。对于客户而言，核心原则简单地意味着低价格。现在我们提供了一部手机，价格为200美元，与iPhone相同，需要与移动运营商签订两年的合同。（200美元现在听起来便宜，但那时手机得到补贴，价格低得多。）我们将价格降到99美元，然后免费提供。没有关系。没有人想要它。

最后，Fire Phone进入市场较晚，而且只有一个运营商AT&T。那时，iPhone服务通过提供许多其他品牌的四个运营商可用。货架上和空中有很多竞争。

如果我们以比iPhone更低的价格点提供Fire Phone，具有大多数功能并承诺Prime，这会产生差异吗？也许。

然而，这个故事的要点是，这个流程提高了成功的可能性，但绝不保证成功。杰夫本人深度参与了Fire Phone的开发。他实际上是PR/FAQ的作者之一，以及项目领导者Ian Freed和Cameron Janes的。他和团队要么相信要么说服了自己他们在创造客户会喜欢的手机，但他们错了。即使是最好的流程也只能改进你的决策质量；没有流程会为你做决定。

确实，Fire Phone的失败并没有导致杰夫质疑创建它的流程。他写道，"我们都知道，如果你全力以赴，你会经常三振出局，但你也会打出一些本垒打。"与棒球不同，在棒球中本垒打最多能带来四分，一个大商业命中可以得分几乎无限数量的得分。关键是理解一小部分非常大的赢家可以支付大量失败或仅谦虚成功的实验。

在Fire Phone撤出后的采访中，杰夫被问及其失败，并回答说，"如果你认为这是一个大失败，我们现在正在进行更大的失败—我不是在开玩笑。"[4]你的发明的规模，因此你的错误，需要与你的组织的增长同步增长。如果不是这样，你的发明可能不足以改变局面。

随着公司变得更大，保持发明机器运转会变得更加困难，一个障碍是"一刀切"的决策制定。在同一份2015年的股东信中，杰夫写道，"一些决定是重大且不可逆转或几乎不可逆转的—单向门—这些决定必须谨慎、仔细、缓慢地、经过深思熟虑和协商的方式做出。如果你走过并且不喜欢你在另一侧看到的，你无法回到你之前的地方。我们可以称这些为第一类决策。但大多数决策不像这样—它们是可改变的、可逆的—它们是双向门。如果你做出了次优的第二类决策，你不必为此承受很长时间的后果。你可以重新打开门并回到。第二类决策可以且应该由高判断力的个人或小团体快速做出。"Prime是一个双向门决策。如果Prime的订阅、免费送货和快速交付的特定组合没有奏效，我们会继续调整公式直到我们做对为止。事实上，Prime不是我们解决这个问题的第一次尝试—它之前是另一个双向门决策，超级节省者送货，最终演变为Prime。另一方面，Fire Phone更多是一个单向门决策：在从市场撤出后，亚马逊没有转身说，"好的，这发生了，现在让我们尝试另一部手机。"

大公司往往会开发一个决策制定流程，该流程设计用于管理单向门决策，正是因为糟糕的决策可能导致大问题，甚至灾难。这个流程通常很慢、庞大且充满风险厌恶。这个流程往往会成为大公司中的主导流程，并且通常被不假思索地应用于双向门决策。结果是速度降低、创意生成受损、创新受阻和开发周期更长。

这就是为什么亚马逊专注于与Day One初创企业相关联的行动速度、灵活性和风险承受心态—同时坚持最高标准。这种方法自公司早期以来一直是亚马逊企业文化的一部分。1999年杰夫写道，"我们必须致力于在每个倡议中不断改进、实验和创新。我们喜欢做先驱者，这是公司的DNA，这是好事，因为我们需要那种先驱精神才能成功。"[5]
